# Contributing to VerifyPulse

Thanks for your interest in contributing! This project is currently in prototype phase.

## How to Contribute

1. Fork the repository
2. Create a feature branch: `git checkout -b feat/your-feature`
3. Make your changes
4. Run tests: `cd backend && python tests/test_clustering.py`
5. Commit with a descriptive message: `git commit -m "feat: description"`
6. Push and create a Pull Request

## Commit Message Format

Use conventional commits:
- `feat:` — New feature
- `fix:` — Bug fix
- `docs:` — Documentation
- `test:` — Tests
- `refactor:` — Code refactoring

## Areas for Contribution

- Adding new news sources (RSS feeds)
- Improving clustering accuracy
- Multilingual support
- Frontend improvements
- Test coverage
- Documentation

## Code Style

- Python: Follow PEP 8
- JavaScript: Use const/let, no var
- Keep functions focused and well-documented
